# Grilling session 001 — post-intake refinement (2026-07-18/19)

Interviewed by the drafting Claude Code session (direct pairing, native
AskUserQuestion); answered by cedric. Trigger: intake of the v0.1
Private Photo Vault spec (`references/intake.md`) flowed directly into
the grill. Scope: the structural fork (one goal vs. map), the map's
destination, and the spec's §14 open decisions — separating user
decisions from executor investigations. Session spanned the 07-18/07-19
boundary; two research runs were dispatched mid-session.

## Q1 — One goal, manual split, or wayfinder map for a 10-phase XXL spec?

Options: Chart a wayfinder map (recommended) / Keep it one goal /
Split manually.

**Answer: Chart a wayfinder map.** Fog is real (KDF benchmarks,
chunk-size tuning, sync design feed later decisions); a map that rolls
forward goal-by-goal beats guessing all tickets upfront or an
unreviewable monolithic goal.

## Q2 — Where is the daily-driver line (the map's destination)?

Options: After local sync, phases 1–5 (recommended) / After playback,
phases 1–2 / After cloud backup, phases 1–7 / Everything, 1–10.

**Answer: Phases 1–5.** Playback polish plus a second synced copy on
own devices gives real redundancy without waiting on the Supabase
stack; cloud backup/sharing become a later map/effort.

## Q3 — Device floor / oldest supported device?

Options: Current devices, latest OS (recommended) / iOS 17-era
hardware / iPhone-only first.

**Answer: (redirected).** Instead of a floor, cedric surfaced a new
requirement: an eventual rsync-like **command-line sync peer** giving
iCloud-like capabilities. Device floor defaulted to latest-current-OS
(recorded as a map note; revisit if an older device joins).

## Q4 — How portable must the core be, given a future CLI peer?

Options: Portable formats + Swift core (recommended) / Rust core with
FFI everywhere / Ignore CLI for v1.

**Answer: Portable formats + Swift core; CLI targets are macOS and
Linux (not Windows), added to the wayfinder map.** VaultCore is a
UIKit/SwiftData-free Swift package; the on-disk/wire formats are the
language-neutral contract (libsodium exists everywhere). Rust/FFI
rejected as toolchain weight before phase 1; ignore-it rejected as a
likely core rewrite.

## Q5 — Signing/deployment for sideloading?

Options: Paid dev account (recommended) / Free Apple ID + weekly
re-sign / Existing paid account.

**Answer: Paid Apple Developer account.** 7-day free-profile expiry is
unacceptable for an app holding the primary copy of photos; 1-year
profiles + easy Vision Pro sideloading.

## Q6 — Chunk addressing scheme (spec §14, must precede CAS code)?

Options: Ciphertext-hash + manifest-level dedup (recommended) / Plain
ciphertext-hash (spec default) / Gallery-scoped convergent.

**Answer: Ciphertext-hash + manifest dedup.** Storage sees only opaque
ciphertext hashes (zero confirmation-attack surface); duplicate
detection happens at import via a plaintext BLAKE2b carried inside
encrypted manifest metadata. Convergent addressing rejected as
must-choose-now migration risk for marginal gain.

## Q7 — DEK epoch rotation in v1?

Options: Defer, reserve epoch field (recommended) / Implement in v1 /
Don't reserve the field.

**Answer: Defer, reserve the field.** Sharing is beyond the
destination, so rotation has nothing to rotate; formats carry an epoch
integer (0) from day one so enabling later needs no migration.

## Q8 — Device trust bootstrap?

Options: Trust-on-first-use (recommended) / Explicit device approval.

**Answer: TOFU.** Within the destination every device is the owner's;
approval ceremonies deferred to a future sharing leg.

## Q9 — Disposition of the two tuning knobs (Argon2id params, chunk size)?

Options: Defaults + in-goal benchmarks (recommended) / Decide values
now / Research first.

**Answer: Research first.** Two Exa deep-research runs drafted and,
after explicit go-ahead (Q10), dispatched:
`argon2id-tuning-on-modern-iphones` and
`chunk-size-for-encrypted-media-cas` (see map research tickets).
In-goal hardware benchmarks still confirm whatever research suggests.

## Q10 — Dispatch which research runs?

Options: Run both (recommended) / Chunk-size only / Argon2id only /
Skip.

**Answer: Run both.** Cheap relative to re-chunking a vault later;
both feed the map's first legs.

## Q11 — Test strategy for crypto + sync?

Options: swift test on core, device for UX (recommended) /
Simulator-first everything / Minimal manual + smoke.

**Answer: swift test on core, device for UX.** Round-trip, tamper,
merge, and two-in-process-peer sync tests run as plain `swift test` on
macOS in VaultCore; simulator/device reserved for playback UX,
benchmarks, and the Multipeer transport.
