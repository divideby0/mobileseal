## Q1 — iOS device-key custody?

Options: Keychain/Secure Enclave (recommended) / passphrase-wrapped
file (spec §5.4 literal).

**Answer: Keychain / Secure Enclave**, device-bound
(WhenUnlockedThisDeviceOnly), no new passphrase UX. The portable
passphrase-wrapped variant ships with the CLI leg via the pluggable
DeviceKeyStore protocol.

## Q2 — Delete UI this leg?

Options: Delete button (recommended) / core-only.

**Answer: yes, iPhone parity** — single delete while viewing +
grid multi-select bulk delete, "exact same behavior as iPhone."

## Q3 — Recently Deleted / delete semantics?

Options: Recently Deleted with restore (recommended) / direct
delete + confirm.

**Answer: Signal/Messages semantics** — _delete for myself_ (soft,
restorable — a per-user state, restorable later from a cloud-sync
menu) vs _delete for everyone_ (real tombstone, only for content I
uploaded — which is precisely the spec §9 author-or-owner validity
rule). Single-user v1 renders this as iPhone-parity Recently
Deleted (soft) with purge emitting the hard tombstone; the
two-button UI surfaces at the sharing legs.
