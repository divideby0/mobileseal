# Grilling session 001 — post-review refinement (2026-07-19)

Interviewed by the drafting Claude Code session (AskUserQuestion,
with two questions answered in chat at cedric's request — he was on
his phone); answered by cedric. Trigger: the pre-grill blind Codex
plan review's mechanical fixes were folded first; this session
resolved the nine user decisions the review and the draft surfaced.

## Q1 — Import scope: PHPicker copy-in, full library, or picker + delete-originals?

Options: PHPicker copy-in only (recommended) / Full library access /
Picker + delete-originals.

**Answer: PHPicker copy-in only** (after a chat explainer with a
delete-after-import user story). Zero photo-library entitlement;
originals stay in Photos — deliberately, since they are the only
redundancy until the sync legs land. Delete-originals ("move into
vault") is tied to the sync milestone and needs PHPhotoLibrary write
auth + Codex A5's post-commit verification protocol.

## Q2 — Backup participation?

Options: Include in backup (recommended) / Exclude / Per-gallery
toggle later.

**Answer: Include.** Ciphertext is safe to back up; vault survives
device migration pre-sync. Gate verifies no `isExcludedFromBackup`
flags.

## Q3 — Face ID convenience unlock?

Options: Defer (recommended) / Include this leg / Never.

**Answer: Defer.** Grounded in the API: biometric caching requires
exporting DEK material, and VaultCore's move-only custody exposes no
key export — needs a designed custody-respecting token API (map
note), not a UI bolt-on. Password-only this leg.

## Q4 — Auto-lock semantics?

Options: Background=lock + 5min idle (recommended) / Grace period /
Explicit-only.

**Answer: Make it a user preference.** Settings surface with
lock-on-background (immediate/grace/off) and idle timeout; strict
recommended values become the defaults. Transient `.inactive` is
always shield-only.

## Q5 — Import fidelity?

Options: Originals + paired Live Photos + full EXIF (recommended) /
Stills-only / Strip location.

**Answer: Originals — plus eventual spatial media support.**
Byte-exact archive; Live Photo pairs stored now (still shown until
Playback); EXIF incl. location kept encrypted. Spatial photos/videos
(MV-HEIC/HEVC) survive automatically as originals; stereo display is
recorded against the future visionOS effort.

## Q6 — Duplicate imports?

Options: Skip with notice (recommended) / Separate entries / Group.

**Answer: Skip with notice.**

## Q7 — Metadata custody for the grid?

Options: Heap + purge-on-lock (recommended) / SecureBytes.

**Answer: Heap + purge-on-lock** — same documented residual class as
decoded pixels; gate-tested purge.

## Q8 — Backgrounding mid-import?

Options: Cancel-and-cleanup + resume prompt (recommended) /
Background continuation.

**Answer: Cancel-and-cleanup** (executor default confirmed).

## Q9 — App identity?

Options: MobileSeal — net.spantree.mobileseal (recommended) / Vault /
Decide at execution.

**Answer: MobileSeal, `com.gmail.cedric.hurst.mobileseal`** (cedric's
own id). Plus a volunteered standing principle: **background
execution may transfer SEALED bytes only; encryption/decryption runs
only while the app is open and unlocked** — recorded at map level for
the sync legs (the two-plane API is the enforcement seam).
